01

Approximation runner

Press to play the 380ms timing in CSS. Useful for verifying the curve feels right (snap-out + overshoot via --ease-rip — the curve B0 reserved exactly for this moment). Production asset will carry far more visual fidelity than CSS keyframes can express; this is timing-only.

NOT THE PRODUCTION ASSET.
CSS keyframes can sketch the timing and the rough shape. The alien-organic quality — noise, soft-edge distortion, the way light pours from a fresh tear — requires a rendered asset. This runner exists to confirm the 380ms feels right against the eye, not to define visual fidelity.

What the eye should see

A held void → a sharp bright flash → a jagged vertical seam opens → an amorphous slime-green mass blooms from inside the seam and pushes the tear edges open → the tear retracts to a faint perimeter glow as the mark settles. No bounce. Overshoot via --ease-rip means the mark crosses its final position before snapping back — a tiny pull, not a wobble.

02

Shape of the rip

Vertical, jagged, center-anchored. Not radial (too superhero-burst), not horizontal (too curtain-open), not diagonal (too slash-the-screen). Vertical tear is the rip's natural axis — inherited from DC-10 / DC-11 visual language (something forcing its way OUT of the void) without lifting recognizable composition.

Rejected shapes (and why)

  • Radial burst — reads as Marvel superhero arrival. Too iconic-in-the-wrong-way; identical to a dozen brand stings.
  • Horizontal slit — reads as a curtain or shutter opening. Domestic, not violent.
  • Diagonal slash — reads as a knife or claw mark. Too violent in the wrong register (aggression-toward-viewer rather than emergence).
  • Multi-slash / cross-tear — too busy; the eye can't lock to a single seam in 380ms.
  • Mark fades in — Rule #9. Not a candidate, never was.

What "jagged" means here

The seam is NOT a clean line. It zigzags ~4–8% horizontally as it descends, like a torn cloth seam or a lightning fork. The jaggedness is what reads as ORGANIC tearing rather than mechanical aperture. In production, the renderer chooses noise frequency, edge thickness, and seam-glow falloff — keep it irregular, keep the edges hot.

What "violent" means here

Force, not aggression. The void doesn't break gently; it gets pried apart by something pushing through. The pre-flash compression is the moment of pressure; the tear is the rupture; the bloom is the mark forcing into space. No screen-shake. No glitch artifacts. Violence is in the motion's CONFIDENCE, not its noise.

Does the void recover?

Yes. By t=380ms the tear has retracted to a faint hairline glow around the mark's perimeter. The void is whole again, but it's marked — the rip leaves a residue, not a wound. On subsequent visits (once-per-session triggered), the page opens in the recovered state. The rip is an event; the static final-frame is the steady state.

03

Timing

380ms total for the rip itself. The window B2 reserved (rip ends → phrase enters at 380ms → cue at 700ms). Easing curve = --ease-rip (the cubic-bezier B0 designated for landing-scale only, never spent in chrome). One curve does the whole rip; the four phases vary only in WHICH ELEMENT moves at which point on the curve.

00 · 0 → 80ms
pre-flash compression
01 · 80 → 180ms
tear opens · flash peaks
02 · 180 → 280ms
mark blooms from seam
03 · 280 → 380ms
tear retracts · mark settles · overshoot

Why 380ms (and not 600 or 200)

200ms is too short — the eye barely catches the bloom; the rip reads as a glitch. 600ms is too long — the visitor starts thinking "loading screen" by 500ms and the rip loses authority. 380ms is the longest interval that still reads as a single event. Comparable to a hard scene cut + breath in film grammar; comparable to a UI snap-toast in chrome.

What happens at t=380ms+

B2's spec inherits. Phrase enters at t=380ms (translate-only, snap easing, ~80ms after mark settles). Scroll cue hairline at t=700ms. Skip affordance was present from t=0 and remains present until the page transitions out of the hero scroll-range.

04

The mark

The thing that rips into being. Formless on purpose. Proposal: the rip-mark IS slime, in its settled form. One mascot, two modes — rip (arrival in B3) and spread (takeover in B4). Flagged as open for the architect's call; the rest of B3 is self-contained either way.

The unification argument

Both deliverables require formless · alien · slime-green · alive. If B3's mark and B4's slime are the same entity at different moments, identity coheres: a visitor sees the mark rip into being, sees the same green presence pour into the space between hero and hotspots, and recognizes ONE thing acting in two modes. Constitution's "Bloomberg-meets-MySpace" balance gets reinforced — there is ONE strange creature that is the platform's avatar; everything else is just the page.

The split argument (for completeness)

If the architect wants them split, the mark becomes an abstract slime-green identity glyph (still formless, but distinct from the mascot). Slime then arrives FRESH in B4 as a separate event. Slightly more identity work, but cleaner separation of concerns. Either path keeps both elements formless and IP-bulletproof.

Either way: what the mark looks like

  • Amorphous — irregular soft-edge blob; border-radius: 48% 52% 56% 44% / 50% 46% 54% 50% baseline, varied per render.
  • Slime-green — radial gradient from oklch(0.92 0.290 148) core to oklch(0.55 0.210 148) edge, anchored on the canon 148° hue.
  • Soft-edged — 2px blur at minimum. NO sharp vector outline. NO eyes. NO body. NO recognizable silhouette.
  • Settles to ~60vmin — the hero rip-slot B1 reserved.
  • Lives lightly — at rest, a sub-perceptible breathing pulse (post-rip, separate from the 380ms event). Spec for B5/B4 to define exact pulse rhythm; B3 only sets the rest shape.
05

Skippable affordance

Discreet glyph in the bottom-right of the hero. Present from t=0 — visible BEFORE the rip starts, not surfacing during. (Surfacing during would mean fading in. Rule #9.) Once dismissed or once-completed, the storage flag fires; subsequent visits skip the rip entirely and open in the final-frame state.

Skip intro esc

Behavior

  • Always present from t=0. Not animated in. Static styled element from initial paint.
  • Click / tap / Esc all jump to final-frame state immediately (mark settled, phrase placed, cue rendered).
  • Once the rip completes naturally OR once the user skips, set the storage flag. Subsequent visits skip the rip entirely.
  • Skip stays visible through the hero scroll-range. As the visitor scrolls past the hero, Skip leaves with the hero — it's a hero-scoped affordance, not a global one.

Storage contract (Dev A wires backend)

Single boolean key. Suggested name: koy.landing.rip.seen. Read at hero mount: if true, open in final-frame state (no rip motion, no skip glyph). If false, play the rip + show skip glyph. Set on rip-completion OR on user-skip OR if reduced-motion / asset-load-fail. Dev A picks the storage backend (sessionStorage vs localStorage vs server-side per-account) based on whatever the auth lane decides. Spec only the contract.

CSS class to ship (B3 → koy-landing.css)

The skip affordance is a new chrome element on landing. Suggested class: .kl-skip. Hairline pill, mono caption, kbd glyph inline. The styling is shown in the demo above; the actual CSS sits in B3's PR to koy-landing.css. Dev A appends; Phase A canon is not touched.

06

Fallback chain

Three failure modes; all resolve to the same fallback — the static final-frame asset, rendered at t=380ms. No transition, no fade, no decoration. The rip is the gift; its absence costs nothing.

Trigger Detection Behavior
prefers-reduced-motion CSS media query, zeroed at root (A1 canon). Open in final-frame state immediately. No rip. No skip glyph (nothing to skip). Storage flag set so subsequent visits behave consistently.
low-bandwidth Spec only the contract: if the motion asset hasn't loaded within 600ms of hero mount, fall back. Detection via simple timeout race. (Network Information API is not universal; Dev A may use it as a hint but timeout is the floor.) Open in final-frame state. Storage flag NOT set — visitor may retry later under better conditions.
asset-load-fail Video / image onerror. Open in final-frame state. Storage flag NOT set.
storage:seen koy.landing.rip.seen === true at mount. Open in final-frame state immediately. No rip (already seen). No skip glyph (nothing to skip).

The final-frame is itself an asset

Per architect: two assets. Production renders the motion artifact AND a static final-frame artifact. The static is used in every fallback path AND in the B1 nav .kl-nav__brand-slot (the small post-rip mark in the top-left chrome). Same source render, different output formats.

Asset coordination + sound
