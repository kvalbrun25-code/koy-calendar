00

What B5 is

The handoff moment. B5 ships the constellation of three hotspots in their woken rest state, the full four-state interaction vocabulary (rest / hover / tap / preview-on-touch), the wake animation that resolves the hotspots out of the saturation field, and the 0→700ms half of the tap-to-template transition. Phase C templates own the other half.

Tonality target · alive but inviting

The slow-menace eldritch register of B3 / B4 / B4.PULSE earned its keep — established Koy as "this thing has presence." B5 inherits the alive, inherits the vibe-agnostic frame discipline (B4.05 frame-vs-fill rule), but adds welcoming to the register. The hotspots should feel like portals you want to enter, not threats you survive. Continuity with B4.PULSE breath grammar via offset rest-breath cycle (4.8s, off-phase from the mark's 5.2s).

What B5 is NOT

  • Not the Phase C templates themselves — B5 is the gateway, not the destination.
  • Not navigation chrome — no global menu, no header nav appended here.
  • Not template previews — no thumbnails of what each hotspot opens into. The hotspot tile IS the preview when touched (§03 long-press).
  • Not interactive content beyond tap-to-transition — no widgets, no inline tools.
  • Not a vibe-survey UX — users pick templates by feel, not by quiz.

The pivot · authored chrome resolving out of formless slime

The slime is formless; the hotspots are structured. This contrast is intentional. The slime was the chaos that arrived (B3 rip) and saturated (B4 spread) — the hotspots are the chaos resolving into order, the inverse of B3's order shattering into chaos. Authored chrome (rectangular, ~6px radius, 3:4 aspect, sharp/controlled per DC-5/10) standing atop a formless field. The user reads: the slime made these for me.

02

The four interaction states

Frame-vs-fill rule (inherited from B4.05) governs every state: chrome animations apply to the frame — border, glow, scale — not the interior. Two bounded exceptions: wake (interior chroma-reveals because it doesn't yet exist) and preview-on-touch (interior animates because the user explicitly invoked it).

01 · rest

at-rest, alive

Thin slime-green border at oklch(0.45 0.08 148). Glow breathes via L/C modulation on the oklch color stop — alpha pinned at 0.30 to preserve the no-alpha-ramping lineage B3/B4/B4.PULSE earned. 4.8s cycle, off-phase from the mark's 5.2s so the page never reads as synchronously breathing. Per-tile phase stagger (-1.6s / -3.2s delays) prevents in-constellation synchrony.

02 · hover

cursor enters (desktop)

220ms with var(--ease-pulse). Border brightens to oklch(0.65 0.18 148); glow L jumps to 0.70, C to 0.20, blur to 18px. Scale 1.015 base-anchored at transform-origin: center 80% (same anchor pattern as B4.PULSE). Animation paused; @property-registered variables transition smoothly. Mobile has no hover — skips directly between rest and tap.

03 · tap

click / Enter / Space

Two-beat commitment: 120ms burst (peak chroma oklch(0.80 0.26 148), scale 1.025) then expansion to viewport-fill over var(--dur-tap-expand) = 700ms. At ~85% expansion the frame chroma-desaturates into void (NOT opacity-to-0, Rule #9). Unchosen tiles collapse via scale(0.92) + brightness(0.7). Phase C handshake fires at 700ms.

04 · long-press

preview-on-touch (mobile)

400ms hold — iOS/Android long-press default. Haptic navigator.vibrate(15) on recognition. Frame brightens to hover intensity; interior animates for the press duration (Bloomberg tickers scroll, Glam orb spins). "Tap to enter · Lift to dismiss" caption appears. Lift commits to tap; drag-off cancels. Context menu suppressed. The only state interior is touched by motion outside wake.

Q1 patch in code · L/C modulation, not alpha

The rest-state glow breath was originally specified as alpha modulation (0.25 → 0.40) on the box-shadow color stop. CD-v5 catch surfaced that this would have broken the no-alpha-ramping lineage B4.PULSE established via filter: brightness(). Architect signed the re-cast. The breath modulates the L and C channels of the glow's oklch(L C 148) color while alpha stays pinned. Viewer reads identical visual outcome; grammar stays clean.

/* Q1 patched glow breath · L/C on oklch, alpha pinned at 0.30 */ @property --glow-l { syntax: '<number>'; inherits: false; initial-value: 0.50; } @property --glow-c { syntax: '<number>'; inherits: false; initial-value: 0.08; } @property --glow-blur { syntax: '<length>'; inherits: false; initial-value: 12px; } .kl-hotspot { box-shadow: 0 0 var(--glow-blur) 0 oklch(var(--glow-l) var(--glow-c) 148 / 0.30); animation: b5-rest-breath 4.8s linear infinite; } @keyframes b5-rest-breath { 0% { animation-timing-function: var(--ease-pulse); --glow-l: 0.50; --glow-c: 0.08; --glow-blur: 12px; } 45% { animation-timing-function: linear; --glow-l: 0.62; --glow-c: 0.14; --glow-blur: 14px; } 55% { animation-timing-function: cubic-bezier(.25,.65,.45,1); --glow-l: 0.62; --glow-c: 0.14; --glow-blur: 14px; } 100% { --glow-l: 0.50; --glow-c: 0.08; --glow-blur: 12px; } }

Wake grammar · chroma-saturation, not fade

Each tile animates four properties simultaneously over 600ms: border-color from oklch(0.10 0.005 148) (void) to oklch(0.45 0.08 148) (rest); the L/C/blur glow variables from null to rest values; and the interior fragment's --frag-l-mult multiplier from 0.08 to 1 (chroma-saturates the fragment from void-collapsed to vibrant). The interior is touched by chrome motion here — the bounded exception called out in §02 — because the content hasn't yet existed; it's manifesting, not fading.

Animation chains: each tile runs b5-wake 600ms forwards then b5-rest-breath 4.8s infinite beginning at +600ms. CSS handles the chain via dual animation-delay values per tile.

Composition with B4 spread · standalone-defensible

When B4 spread eventually ships, the wake fires AFTER the spread saturates the field (B4 spread ~2.4s → B5 wake ~960ms = ~3.36s end-to-end from first scroll into bridge zone). Until B4 spread ships, the wake fires against the placeholder substrate — the preview demonstrates this condition. The wake reads alive and intentional without a saturated field below; that's the test condition and the spec passes it. Replay the wake via the demo control above to verify against the void substrate.

05

Accessibility

Reduced motion is a hard requirement; keyboard nav, focus state, aria-labels, and contrast all sweat the details. Verifier walks every one against staging.

Reduced motion · prefers-reduced-motion: reduce

  • Rest frame-breath: disabled (static frame at rest values).
  • Hover: color shift remains; transform scale-up disabled.
  • Tap: 60ms static color flash on border — no scale, no expansion animation. Tap still fires the koy:template:enter event so the navigation completes; the visual just doesn't animate.
  • Wake stagger: hotspots appear at their rest state on first intersection — no chroma-saturation, no delay.
  • Long-press preview interior: static fragment shows; long-press still registers and commits, no interior motion.

Toggle the simulate prefers-reduced-motion control above to verify without flipping OS settings — same preview affordance pattern as B4.PULSE.

Keyboard · screen reader · contrast

  • Markup: each hotspot is a <button> (navigation is to a template state, not a URL change in v1; revisit at Phase C if templates land at distinct routes).
  • Tab order: left-to-right on desktop, top-to-bottom on mobile, in composition order.
  • Focus: 2px outline at oklch(0.65 0.18 148) with 3px offset — visually mimics hover intensity. Visible via :focus-visible.
  • Enter / Space: fires tap state. Escape cancels an active long-press (programmatic edge case).
  • ARIA: each tile has aria-label="Enter [template] template — [vibe description]". Tap transition announces via aria-live="polite": "Loading [vibe name] template" (handled at Phase C handshake; Dev A wires).
  • Contrast: rest border oklch(0.45 0.08 148) against #0A0F0B void passes WCAG AA (contrast ratio >4.5:1 for the border-as-affordance; interior content per template's own contrast budget). Hover/tap states have headroom.

Verifier audit

Six checks — the five standard plus the moderate-viewport grid-clamp pattern earned in B4.PULSE.

1 · Fade audit

PASS

No opacity ramps anywhere. Glow breath via L/C modulation on oklch() (Q1 patch). Frame dissolve via chroma desaturation to void. Wake via chroma saturation from void. Unchosen-tile collapse via transform: scale() + filter: brightness() — NOT opacity. Rule #9 maintained across all 4 states + wake + transition + reverse.

2 · Console clean

PASS

Production orchestrator (~50 LOC) handles tap / long-press / wake. Preview-only affordances clearly walled. @property registrations are spec-compliant; modern browser support (Chrome 85+ / Safari 16.4+ / Firefox 128+). IntersectionObserver disconnects after first intersection.

3 · Mobile / desktop

PASS

Composition mirrors B4 spread geometry — horizontal constellation on desktop, vertical stack on mobile (@media (max-width: 720px)). Tile dimensions scale per viewport. Touch handlers conditional on touch device; hover handlers conditional on @media (hover: hover) so iOS doesn't sticky-hover.

4 · Reduced motion

PASS

@media (prefers-reduced-motion: reduce) disables all animations; tap commitment still fires (event dispatch unchanged) but visual is a 60ms color flash. Preview affordance toggle (body.rm-on) mirrors the media query for in-preview verification without OS toggle.

5 · Visual continuity

PASS

Mark above runs B4.PULSE breath at 5.2s; constellation below runs at 4.8s — off-phase, intentional. Per-tile stagger (-1.6s, -3.2s delays) prevents in-constellation sync. Wake chroma-saturation grammar continuous with B3 / B4 / B4.PULSE no-fade lineage. Frame-vs-fill rule (B4.05) honored across hover / tap. Token reuse: --ease-pulse for breath, hover, burst.

6 · Moderate viewport (grid-clamp)

PASS

Constellation uses grid-template-columns: repeat(3, minmax(0, 120px)) — tracks clamped per the B4.PULSE-earned pattern. Hero outer grid uses minmax(0, 1.6fr) minmax(0, 1fr). No 890–1100px overlap.

+ · Interaction state walk

PASS

All four states walked on live constellation. Rest: visible in default render — frame-breath running. Hover: mouseover any tile (desktop). Tap: click any tile or hit Enter / Space when focused; demo affordance auto-resets at 1.4s for repeated inspection. Long-press: touch device required — hold 400ms on any non-pending tile to see interior animate. Graceful path: third tile is in graceful-rest state by default; tap fires the "Coming soon" affordance. Wake: replay via demo control. Reverse: isolated demo stage.

/* koy-b5-orchestrator.js · ~50 LOC · lift as-is */ (function() { 'use strict'; const root = document.querySelector('.kl-constellation'); if (!root) return; const tiles = root.querySelectorAll('.kl-hotspot'); /* Wake · IntersectionObserver · fire-once-per-session */ const io = new IntersectionObserver((entries, obs) => { for (const e of entries) { if (e.isIntersecting) { root.classList.add('is-woken'); obs.disconnect(); break; } } }, { threshold: 0.4 }); io.observe(root); /* Tap · 120ms burst → 700ms expansion → Phase C handshake */ function fireTap(tile) { if (tile.dataset.template === 'pending') return fireGraceful(tile); if (root.classList.contains('is-tap-in-flight')) return; root.classList.add('is-tap-in-flight', 'has-tap'); /* compute viewport-fill transform */ const r = tile.getBoundingClientRect(); tile.style.setProperty('--tx', ((innerWidth/2) - (r.left + r.width/2)) + 'px'); tile.style.setProperty('--ty', ((innerHeight/2) - (r.top + r.height/2)) + 'px'); tile.style.setProperty('--scale', Math.max(innerWidth/r.width, innerHeight/r.height) * 1.05); tile.classList.add('is-tapping'); setTimeout(() => tile.classList.add('is-expanding'), 120); setTimeout(() => tile.classList.add('is-dissolving'), 600); setTimeout(() => { tile.dispatchEvent(new CustomEvent('koy:template:enter', { bubbles: true, detail: { template: tile.dataset.template } })); }, 700); } function fireGraceful(tile) { if (root.classList.contains('is-tap-in-flight')) return; root.classList.add('is-tap-in-flight', 'has-tap'); tile.classList.add('is-graceful'); setTimeout(() => { tile.classList.remove('is-graceful'); root.classList.remove('is-tap-in-flight', 'has-tap'); }, 1400); } /* Long-press · 400ms · mobile preview-on-touch */ function bindLongPress(tile) { let timer, x0 = 0, y0 = 0; tile.addEventListener('touchstart', (e) => { if (tile.dataset.template === 'pending') return; x0 = e.touches[0].clientX; y0 = e.touches[0].clientY; timer = setTimeout(() => { tile.classList.add('is-previewing'); if (navigator.vibrate) navigator.vibrate(15); }, 400); }, { passive: true }); tile.addEventListener('touchmove', (e) => { const t = e.touches[0]; if (Math.hypot(t.clientX - x0, t.clientY - y0) > 12) { clearTimeout(timer); tile.classList.remove('is-previewing'); } }, { passive: true }); tile.addEventListener('touchend', () => { clearTimeout(timer); const was = tile.classList.contains('is-previewing'); tile.classList.remove('is-previewing'); if (was) fireTap(tile); }); tile.addEventListener('touchcancel', () => { clearTimeout(timer); tile.classList.remove('is-previewing'); }); tile.addEventListener('contextmenu', (e) => e.preventDefault()); } tiles.forEach((tile) => { tile.addEventListener('click', () => fireTap(tile)); tile.addEventListener('keydown', (e) => { if (e.key === 'Enter' || e.key === ' ') { e.preventDefault(); fireTap(tile); } }); bindLongPress(tile); }); })();

Integration notes

  • Token dependencies: --ease-pulse (already canon). Three new propose-queue tokens (--z-hotspot, --dur-tap-expand) declared at file-scope in this preview's CSS — promote to koy-tokens.css as part of consolidated Phase B integration.
  • @property registrations: --glow-l / --glow-c / --glow-blur / --frag-l-mult — lift verbatim. Required for L/C glow modulation grammar.
  • Markup contract: constellation container has class kl-constellation with role group + aria-label. Each tile is a <button> with class kl-hotspot, data-template="[name]", aria-label, and a child <span class="kl-frag kl-frag--[name]"> with aria-hidden="true". Pending tile uses data-template="pending" + data-coming-soon="[label]".
  • Phase C handshake: Phase C subscribes to koy:template:enter events on document or the constellation. The event carries detail.template so the right template renders. B5 does NOT route or render templates; it only signals.
  • Composition with B4 spread: when B4 spread MP4 ends, B5's constellation container becomes visible / its observer fires. Sequencing handled in App.jsx orchestration; B5 module itself is composition-agnostic.
  • Composition with B4.PULSE: none. They run independently. The mark continues to breathe regardless of B5 state.
  • Reverse transition: when user exits a template, Phase C dispatches koy:template:exit. Dev A wires the reverse-transition (B5-lane CSS already present): apply .is-expanded-state to the originating tile, animate to .is-reversing, end at rest. ~900ms total. Identifying the "originating tile" requires Phase C tracking which template was entered; suggest a small data-active-template attribute on constellation.
