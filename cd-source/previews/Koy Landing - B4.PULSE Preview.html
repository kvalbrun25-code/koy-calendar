Koy Landing / pulse

B3 v2 ends with the mark settled. Without a pulse the mark reads as a screenshot — frozen, finished, dead. B4.PULSE breathes the mark in place, the same logic as a sleeping animal's chest rising and falling. Held but alive; held but not finished; held but watching. Pure CSS keyframe on the static B3 v2 final-frame asset. No Seedance dependency. Independent ship from the parked B4 spread — the cheapest deliverable in Phase B, biggest perceptual lift per agent-cycle.

00

What B4.PULSE is

An ambient at-rest loop on the settled mark. Two synchronized effects (scale + brightness), optionally a third (blur), running on a 5.2s cycle — slower than human breath, calibrated subliminal. The viewer should never consciously notice the breath; only register the mark as alive.

Tonality target

Alive but held. Pregnant with motion. Slow-menace at rest. Same eldritch register as B3 v2 — the mark hasn't lost its presence after the rip, it's just at rest. Body grammar carried forward, not reset. The breath answers one viewer question: “Wait, is this thing still there?” The breath says yes, without moving.

What B4.PULSE is NOT

  • Not a separate motion sequence with its own arc — it's ambient, looping, internal.
  • Not a ripple or wave — those imply external disturbance. The breath is internal.
  • Not the slime spread (B4 proper, Seedance, parked). The spread happens BELOW the mark; the pulse continues uninterrupted ON the mark.
  • Not interactive — the viewer doesn't trigger it; it just is.
  • Not attention-grabbing — subliminal calibration is load-bearing. Conscious noticing = failed spec.

Three modes, one entity (continuity with B3 v2 canon)

B3 settled the slime into a landing-resolved form. B4.PULSE doesn't introduce a fourth mode — it expresses the landing-resolved mode's at-rest condition. The mark is still in mode 2; it's just breathing. When B4 spread eventually ships, the spread is the slime's spread-formless third mode acting BELOW; the mark above continues to breathe in landing-resolved mode throughout. No coupling, no synchronization — two simultaneous motions on different surfaces.

Anchor · transform-origin: center 80% · v2 geometry

The B3 v2 settled mark is a vertical pour with a widened base — denser at the bottom, narrower above (locked geometry per CD v4 §00 Shift 4). Scaling from the geometric center would push the mass equally in all directions; scaling from center 80% Y means the densest part of the body — the base — stays planted while the upper mass drifts subtly skyward at peak. This reads as “the mass breathes from its center of gravity” rather than “the mass zooms toward the viewer.” Spring-not-drain reading reinforced. transform-origin is relative to the mark element's own box, not the page.

Brightness asymmetry · 0.94 trough is intentional

The 0.94 floor (slightly dimmer than canonical 1.0) is the “settles into its shadow” beat at the bottom of the exhale. Canonical brightness sits roughly at the 50%-of-progress mark of either segment, NOT at the loop boundary. Production-side continuity note for Dev A: for the seamless MP4→static handoff, the B3 v2 MP4's final frame should match the pulse's trough state (brightness ~0.94, scale 1.000) so the moment of transition lands on the breath's natural low. Seam reconciliation between MP4 final frame and static asset is Dev A's lane (per architect Q5 sign); noted here only as awareness, not as a B4.PULSE scope item.

Composition · keyframe block as authored

/* the deliverable: one class + one @keyframes block */ .kl-mark-rest { animation: b4-pulse 5.2s linear infinite; transform-origin: center 80%; will-change: transform, filter; /* hint compositor */ } @keyframes b4-pulse { 0% { animation-timing-function: var(--ease-pulse); /* inhale */ transform: scale(1.000); filter: brightness(0.94) blur(0px); } 45% { animation-timing-function: linear; /* peak hold */ transform: scale(1.012); filter: brightness(1.06) blur(0.4px); } 55% { animation-timing-function: cubic-bezier(.25,.65,.45,1); /* exhale */ transform: scale(1.012); filter: brightness(1.06) blur(0.4px); } 100% { transform: scale(1.000); filter: brightness(0.94) blur(0px); } } @media (prefers-reduced-motion: reduce) { .kl-mark-rest { animation: none; filter: none; } }

CD's read · include the blur

The two cells run side-by-side; the substrate mark is already filter: blur(3px) baseline (matching the B3 v2 rebuild). The breath's additional 0→0.4px on top is small enough that it doesn't add visible “defocus” — it adds a soft edge breathing that reads as the body relaxing outward at peak inhale. Without blur, scale+brightness alone produces a slightly more mechanical pulse — the edges remain pixel-stable while the body grows, which reads as a UI element scaling, not a body breathing.

The risk the architect flagged (“looks like a render artifact”) doesn't materialize at this magnitude. Default: include. If founder/architect read it as muddy on production substrate (real JPG vs CSS rebuild), omit is a one-line patch (drop the blur term from filter).

04

Verifier audit

Five checks per CD-lane discipline. Status pills are CD's own pre-handoff assessment; Claudette runs the same audit against staging URL after Dev A integrates.

1 · Fade audit

PASS

Constitution Rule #9 prohibits the fade primitive. B4.PULSE animates transform (scale) and filter (brightness, blur). No opacity in the keyframes. No fade-in / fade-out on the mark. Brightness is a CSS filter channel, not opacity — it modulates rendered luminance, never alpha.

2 · Console clean

PASS

Pure CSS deliverable — one @keyframes, one class, one media query. Zero JS for the pulse itself. Preview file contains one tiny rm-toggle button handler (preview-only affordance to simulate the reduced-motion path without OS toggle); production integration carries no JS.

3 · Mobile / desktop

PASS

transform + filter are GPU-accelerated on iOS Safari, Chrome Android, and all desktop targets. Compositor-thread — no layout, no paint. Cost is effectively free on low-end devices. Mobile read: subliminal at small viewport — the breath should never feel like a UI element pulsing for attention.

4 · Reduced motion

PASS

@media (prefers-reduced-motion: reduce) sets animation: none; filter: none; — the mark renders at its canonical static state (no scale, no brightness modulation, no blur breath). Test affordance in §01 (data-rm="on") lets verifier confirm without toggling OS preference. Hard requirement satisfied.

5 · Visual continuity with B3 v2 settled state

PASS

The B3 preview's .rip-mark-blob at phase-id=3 is transform: scale(1) with no filter modulation — a static slime body. The B4.PULSE substrate (CSS rebuild here, real JPG in production) loops past scale(1.000) at the trough; the body grammar reads continuous. The two are the same mark in the same mode. The pulse adds at-rest breath, not a new visual identity. Production seam (MP4 last frame → static asset) is Dev A's reconciliation; the pulse contract is verified against the static substrate.

The two snippets

JSX — class applies to the static final-frame <img> */ <img src="/assets/koy-rip-static-final-frame-v2.jpg" alt="" className="kl-mark-rest" aria-hidden="true" />

Integration notes for Dev A

  • Token dependency: var(--ease-pulse) is declared in koy-tokens.css. No new tokens. No new dependencies.
  • If the existing image element already has transform: translate(…) for positioning, layer it: the pulse keyframe writes transform: scale(...) which would clobber translate. Two options: (a) wrap the image in a positioning <div> and apply .kl-mark-rest to the image, (b) write the static translate into --mark-tx/--mark-ty and prepend in each keyframe transform. The preview file uses option (a) implicitly — pulse runs on a freshly-positioned element.
  • MP4→static seam (Dev A's lane per Q5): MP4's final frame should land at the pulse's trough state (brightness ~0.94, scale 1.000). The onended handler should swap MP4 for static and apply .kl-mark-rest in the same tick. The pulse begins at 0% of its keyframe block by default — which is the trough. Seamless handoff comes for free if the MP4's last frame matches the trough render.
  • Reverse interaction with B3's seen-flag: if koy.landing.rip.seen === true, the static asset renders at mount (no rip motion). Apply .kl-mark-rest at mount in that branch too — the mark should breathe from page load, not only after MP4 ended.
